Create an ad-hoc or Internet access point (hotspot) with Virtual WiFi

To create a Wi-Fi network hosted by the wireless adapter card of your computer you must first enter a hotspot name and a password, and then choose the desired type of hotspot by selecting either the name of the Internet connection you want to share or the option 'Without Internet connection (ad-hoc)'.

If there are multiple connections available then an auxiliary option named 'Detect the fastest Internet connection' will be shown in the list with connections - which detects and automatically select the best Internet connection available. To start the Wi-Fi hotspot with the specified characteristics press the 'Start Wi-Fi' button.

Once the hotspot is active, you can stop it by pressing 'Stop Wi-Fi' button, see some internal details about the hosted wireless network by pressing the button 'Details' or see a list with the devices connected to the created hotspot in the 'Connected clients' tab.

In the 'Details' window will be shown the maximum number of clients that can be connected simultaneously, the BSSID (the MAC address of the wireless adapter and also of the virtual router), the channel, the password used and other information. To view the password you can also press the eye icon located near the password field, on the 'Configuration' tab.

See the devices connected to the hotspot

By accessing the 'Connected clients' tab you will see all the devices that are connected to the hotspot or are in the process to do so. If the option 'Enable auto-refresh of the connected clients list' is checked in the 'Settings' tab, the devices are automatically detected, otherwise you must press the refresh button to see the updated list of connected clients.

When a device is listed with more than one IP address, means that that device was recently connected with a different IP address. The old IP address will disappear after a couple of minutes.

Virtual WiFi Settings options

In the 'Settings' tab you can select the language of the application, opt to automatically start the application with Windows, keep the hotspot active at the application exit, check for updates at every start-up, activate auto refresh of the clients list or enable manual assignation of the internal IP address used to host the hotspot (the IP address is usually 192.168.137.1 ). You can also revert to the original settings by pressing the button Reset Wi-Fi settings while there is no hotspot momentarily running.



Important notes:

Auto assignation of the IP address is disabled as default, the enabling should be done only in case in which the hotspot is successfully created but the devices cannot connect to it - the problem being that Windows cannot assign a valid IP address to the virtual hotspot, thereby the resulting connection will be either unavailable or will be without Internet access. If you opt to activate this option, a backup with the initial settings will be created, so the disabling will cause the restoration of the initial settings.

If the option 'Enable auto-refresh of connected clients list' is checked, the program will use additionally processor resources - in most of the cases that ain't a problem (it will use below 0.5% of processor).

If the option 'Keep the hotspot active when exit' is enabled, and implicitly is, then, if a hotspot is running and you press the exit button, the application will be terminated but the hotspot will continue to be activated. You can stop the wifi hotspot by reaccessing the program, case in which the program detects the running hotspot and show the stop button. Also, the shutdown of the computer while the hotspot is running will cause, of course, the disabling of the hotspot, with no implication for the next start up, unless the option 'Start saved Wi-Fi with program' from 'Configuration' tab and 'Run on Windows Startup' from 'Settings' tab are enabled.
